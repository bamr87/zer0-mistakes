# Security Policy

## Supported Versions

The following versions of Zer0-Mistakes receive security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| 0.22.x  | :white_check_mark: |
| < 0.22  | :x:                |

## Reporting a Vulnerability

To report a security vulnerability, please **do not** open a public GitHub issue.

**Contact:** Open a [GitHub Security Advisory](https://github.com/bamr87/zer0-mistakes/security/advisories/new) (private disclosure). If you are unable to use that, email the maintainers via the contact information in the repository profile.

**What to include:**
- Description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Affected versions
- Any suggested mitigations (optional)

**Response timeline:**
- Acknowledgement within **48 hours**
- Assessment and plan within **7 days**
- Patch release within **30 days** for critical issues, **90 days** for lower severity

We follow responsible disclosure: we will coordinate a fix and credit the reporter before making any public disclosure.
