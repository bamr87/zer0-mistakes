# MDX with Tailwind CSS

This page demonstrates MDX (Markdown + JSX) capabilities with Tailwind CSS styling in the zer0-mistakes theme.

## What is MDX?

MDX allows you to use JSX components directly in your markdown files, enabling:

- **Interactive components** within static content
- **Reusable UI elements** across multiple pages
- **Dynamic content** with React-like syntax
- **Custom styling** with Tailwind CSS utilities

## Basic Markdown Features

MDX fully supports standard markdown syntax:

### Lists

- Item one with **bold text**
- Item two with *italic text*
- Item three with `inline code`

### Code Blocks

```javascript
// JavaScript code example
const greeting = "Hello, MDX!";
console.log(greeting);
```

## Responsive Design

Tailwind's responsive utilities work seamlessly:

<div className="tw-hidden md:tw-block tw-bg-green-100 tw-p-4 tw-rounded tw-mb-4">
  <p className="tw-text-green-800">This box is hidden on mobile and visible on medium screens and up.</p>
</div>

<div className="tw-block md:tw-hidden tw-bg-yellow-100 tw-p-4 tw-rounded tw-mb-4">
  <p className="tw-text-yellow-800">This box is visible on mobile and hidden on medium screens and up.</p>
</div>

## Best Practices

1. **Use the `tw-` prefix** for all Tailwind classes to avoid conflicts with Bootstrap
2. **Combine frameworks wisely**: Use Bootstrap for layout and navigation, Tailwind for custom styling
3. **Keep it semantic**: Use proper HTML elements with utility classes for styling
4. **Test responsiveness**: Check your content on different screen sizes
