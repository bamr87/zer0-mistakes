{% comment %} =================================================================== SETUP CHECK - Detect unconfigured site state =================================================================== File: setup-check.html Path: _includes/components/setup-check.html Purpose: Provides Liquid variables to detect whether the site has been configured by the user or is still running on theme defaults. Usage: {% include components/setup-check.html %} {% if site_needs_setup %} ... show setup guidance ... {% endif %} Detection Logic: The include sets site_needs_setup to true when the site appears to be running with default/unconfigured values. It checks: 1. Explicit site_configured: true flag (fastest path) 2. Whether the title is still the theme default 3. Whether founder/name are still theme defaults Output Variables: - site_needs_setup : true if the site hasn't been personalised - site_is_user_repo : true if this appears to be a .github.io repo is not set explicitly) ── Treat the site as "needs setup" if any of these are true: • No founder/author/email configured • Title is empty or matches a known placeholder • The site has no content
