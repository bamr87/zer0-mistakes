{%- comment -%} =================================================================== AUTHOR AVATAR URL RESOLVER =================================================================== Resolves an author's avatar image URL from an author data hash. Pass the author entry (from _data/authors.yml) as `data`. Resolution order: 1. `avatar` is a full URL (contains "://") — used as-is. This is how you reference an external avatar, e.g. a GitHub one: avatar: "https://avatars.githubusercontent.com/u/10567847?v=4" 2. `avatar` is a relative path (e.g. /images/authors/me.png) — prefixed with the site's assets path ({{ site.baseurl }}/{{ site.public_folder }}). 3. No `avatar`, but a `github` handle is set — falls back to that account's GitHub avatar: https://github.com/.png 4. Otherwise — empty output (the caller renders a fallback icon). Output is the bare URL with no surrounding whitespace. Capture it (the example below is brace-free so this comment does not self-execute): capture avatar_url -> include components/author-avatar-url.html data=author_data =================================================================== {%- endcomment -%} {%- assign _avatar = include.data.avatar -%} {%- if _avatar and _avatar != "" -%} {%- if _avatar contains "://" -%}{{ _avatar }}{%- else -%}{{ site.baseurl }}/{{ site.public_folder }}{{ _avatar }}{%- endif -%} {%- elsif include.data.github and include.data.github != "" -%} https://github.com/{{ include.data.github }}.png {%- endif -%}
