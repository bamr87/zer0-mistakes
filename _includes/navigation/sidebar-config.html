{%- comment -%} Component: sidebar-config Path: _includes/navigation/sidebar-config.html Purpose: Resolve the effective left-sidebar configuration for the current page. Jekyll `assign`s inside an include propagate to the including template, so this is the single source of truth shared by _layouts/default.html, _includes/core/header.html, navigation/sidebar-left.html and navigation/unified-drawer.html (previously each duplicated this logic and drifted). Params: none — reads page / collection / site state. Sets (caller scope): sidebar_enabled — false when the page opts out entirely (`sidebar: false`, or featured/breaking post_type with no explicit setting) sidebar_nav — resolved concrete mode: "collection", "categories", "tags", or the name of a _data/navigation/*.yml file (nil = none) sidebar_has_content — true only when sidebar_nav will actually render items (gates the left column and the mobile toggle) sidebar_title — heading text for the sidebar panel sidebar_icon — Bootstrap Icons class for the heading Config precedence (most specific wins): 1. page.sidebar (false | true | {nav, title, icon, ...}) 2. `sidebar:` metadata on the page's collection (_config.yml `collections.
