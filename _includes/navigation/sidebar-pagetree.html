{%- comment -%} Component: sidebar-pagetree Path: _includes/navigation/sidebar-pagetree.html Purpose: "pages" sidebar mode — a live URL-hierarchy tree of the site's pages under a base path, grouped by their first path segment into collapsible sections. Zero curated _data/navigation/*.yml file needed: the tree is derived from page permalinks, so it stays in sync with the content automatically. Use it for flat page trees that aren't a Jekyll collection (the "collection" mode covers collections). Example: a docs area whose pages live at /docs/
//. Params (include.* override the page.sidebar hash): base (required) URL prefix to root the tree at, e.g. "/docs/wargames/". Only pages whose url is under this prefix are listed. order_by (optional) front-matter key to sort each section's pages by. Jekyll `sort` is numeric-aware, so a numeric field like `nav_order` yields natural (0,1,2,…,10) order. Default: url (path order). expand (optional) true to expand every section (default: only the section containing the current page). Front matter (per page, all optional): sidebar_exclude: true — hide this page from the tree. sidebar_label: "…" — override the link text (else title). Heading: title/icon resolved by sidebar-config.html (page → site). Depends on: site.html_pages, Bootstrap 5 collapse, Bootstrap Icons, the nav-tree styles (_sass/core/_nav-tree.scss). Emits the same nav-tree markup as nav-tree.html / sidebar-folders.html so active-state, collapse and styling behave identically. {%- endcomment -%} {%- assign _base = include.base | default: page.sidebar.base -%} {%- if _base -%} {%- assign _order_by = include.order_by | default: page.sidebar.order_by -%} {%- assign _expand_all = page.sidebar.expand | default: false -%} {%- comment -%} Pool = plain pages + every collection document, so the tree works for flat pages (site.html_pages) AND collection content (site.documents) alike.
