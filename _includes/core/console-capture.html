{%- comment -%} =================================================================== CONSOLE CAPTURE — early, tiny console/error ring buffer =================================================================== File: console-capture.html Path: _includes/core/console-capture.html Purpose: Capture a capped, in-memory buffer of console output and uncaught errors, installed HIGH in — before the theme's deferred scripts run — so the page-feedback widget (assets/js/page-feedback.js) can attach the logs and JS errors that led up to a report. Renders: only when page_feedback.enabled AND page_feedback.capture_logs. Privacy: in-memory only, never persisted, never sent anywhere until the user opens the feedback modal, reviews the captured logs, and chooses to submit. Any single entry is truncated; the buffer is a fixed-size ring (page_feedback.log_limit). Depends on: nothing — pure inline ES5, GitHub Pages-safe (no CDN). =================================================================== {%- endcomment -%} {%- if site.page_feedback.enabled and site.page_feedback.capture_logs -%} {%- endif -%}
