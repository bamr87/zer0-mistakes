{% comment %} Feature: ZER0-078 {% endcomment %} {%- comment -%} Component: i18n Path: _includes/core/i18n.html Purpose: Resolve the active UI-string set into the `ui` variable, per-page-language aware. English strings come from _data/ui-text.yml (the human-maintained source of truth); alternate languages come from _data/i18n/.yml, which is GENERATED by scripts/translate.rb — never hand-edited. Params: lang (string, optional) — overrides page.lang. REQUIRED when the caller is rendered via {% include_cached %} (pass lang=page.lang there) so each language gets its own cache entry instead of sharing one page-invariant render. Depends on: _data/ui-text.yml, _data/i18n/.yml (generated), site.translation.* config Notes: Liquid assigns made inside an include are visible to the caller, so after {% include core/i18n.html %} simply use {{ ui.some_key | default: "English fallback" }}. Missing languages or keys degrade to English/literal defaults. {%- endcomment -%} {%- assign _i18n_lang = include.lang | default: page.lang | default: site.translation.source_lang | default: site.locale | default: 'en' -%} {%- assign _i18n_code = _i18n_lang | slice: 0, 2 | downcase -%} {%- assign _i18n_source = site.translation.source_lang | default: 'en' -%} {%- assign ui = site.data.ui-text[site.locale] | default: site.data.ui-text.en -%} {%- if _i18n_code != _i18n_source -%} {%- assign _i18n_generated = site.data.i18n[_i18n_code] -%} {%- if _i18n_generated -%} {%- assign ui = _i18n_generated -%} {%- endif -%} {%- endif -%}
