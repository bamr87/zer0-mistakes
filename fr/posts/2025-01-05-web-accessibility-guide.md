L'accessibilité web n'est pas seulement une exigence légale — il s'agit de créer des expériences inclusives pour tous. Ce guide couvre les pratiques d'accessibilité essentielles pour le développement web moderne.

## Pourquoi l'accessibilité est importante

Plus d'un milliard de personnes dans le monde vivent avec une forme de handicap. Rendre votre site web accessible :

- **Élargit votre audience** : touchez plus d'utilisateurs
- **Améliore le SEO** : les sites accessibles sont mieux référencés
- **Renforce l'utilisabilité** : bénéfique pour tous les utilisateurs
- **Garantit la conformité** : répond aux exigences légales

## Principes fondamentaux (POUR)

### 1. Perceptible

Les utilisateurs doivent pouvoir percevoir l'information :

- Fournir des alternatives textuelles pour les images
- Proposer des sous-titres pour les vidéos
- Assurer un contraste de couleur suffisant
- Rendre le contenu adaptable

### 2. Utilisable

Les utilisateurs doivent pouvoir utiliser l'interface :

- Accessible au clavier
- Aucun contenu provoquant des crises
- Structure navigable
- Suffisamment de temps pour accomplir les tâches

### 3. Compréhensible

Le contenu doit être compréhensible :

- Texte lisible
- Comportement prévisible
- Aide à la saisie
- Prévention des erreurs

### 4. Robuste

Le contenu doit fonctionner avec diverses technologies :

- HTML valide
- Nom, rôle et valeur pour les composants
- Messages de statut annoncés

## Gains rapides

1. **Utilisez un HTML sémantique** : `<nav>`, `<main>`, `<article>`
2. **Ajoutez du texte alternatif** : décrivez les images de manière pertinente
3. **Vérifiez le contraste** : utilisez des outils comme WebAIM
4. **Testez la navigation au clavier** : parcourez votre site avec la touche Tab
5. **Ajoutez des étiquettes ARIA** : là où le HTML sémantique ne suffit pas

Commencez dès aujourd'hui à rendre le web plus accessible !
